## 3. Бот учебки

```text

```

### Лист `Траскрибация`

Важно: фактическое имя листа может быть **`Траскрибация`** с опечаткой. В коде надо использовать фактическое имя листа.

Структура:

| Колонка | Значение |
|---|---|
| `Ключ` | ключ кандидата или session_key |
| `Роль` | строка с `ROLE-ID | компания | ФИО` |
| `Дата` | дата диалога |
| `Текст` | полная транскрибация |

Правило парсинга роли:

```js
roleId = roleText.split('|')[0].trim()
```

### Лист `Роли`

Нужен для портрета клиента.

Из него подтягивать:

- ID роли;
- ID команды;
- название компании;
- ФИО клиента;
- должность;
- СНО;
- взаимодействия с клиентом ранее;
- информация о клиенте;
- род деятельности;
- опыт ведения бизнеса;
- форма организации;
- критерий успеха;
- критерий провала;
- целевое действие;
- возражения;
- тон;
- дополнительная информация.

### Роль в новой системе

```text
training_bot_dialogs + training_bot_roles + training_bot_analysis
```

## 4. Вопросы для автомануала (К)

Google Sheet:

```text
Вопросы для автомануала (К)
```

Используется для AI-оценки звонков по мануалу.

Главный лист:

```text
Базовый мануал
```

Важные колонки:

```text
Финальная формулировка вопроса_20.04.2026
Инструкция для модели / Комментарий_20.04.2026
```

Также важен лист:

```text
Сегментный мануал
```

Там нужно использовать финальные формулировки и инструкции по сегментам.

### Как использовать

Оценка звонка должна быть двухслойной:

1. **Базовый мануал** — общий для всех звонков.
2. **Сегментный мануал** — зависит от сегмента клиента/новичка.

Блоки базового мануала:

- установление контакта;
- выявление потребностей;
- презентация продукта;
- работа с возражениями;
- закрытие.
