# Зафиксированные решения и открытые вопросы

## Зафиксированные решения

### 1. Google Sheets не должен быть ядром

Решение:

```text
Сервер = источник истины
Google Sheets = import/export/legacy source
```

Причина:

- Google работает нестабильно;
- формулы хрупкие;
- одна строка не подходит для многоэтапной аналитики;
- сложно строить dashboard и сравнения.

### 2. Base key и session key разделяются

Решение:

```text
base_key = человек
session_key = этап/звонок/попытка
```

Пример:

```text
MDLK5Y
MDLK5Y-OB-01
MDLK5Y-EDO-01
MDLK5Y-REAL-S-01
```

### 3. Основной ключ новичка не заменяется

`Кандидаты.Ключ` / `base_key` остаётся постоянным.

Дочерние ключи живут отдельно.

### 4. Codex используется только для сервера

GPT делает документы, проектирование, Google Drive/Sheets, структуру, ТЗ.

Codex делает сервер, API, БД, logs, deploy, MCP.

### 5. Первый продукт — карточка новичка, не dashboard

Dashboard строится после нормализации данных.

Первый MVP:

```text
форма сборки карточки → normalized JSON → AI-анализ → сохранение → отчёт
```

### 6. Бот учебки — важный источник

Нужно импортировать:

- `Траскрибация`;
- `Роли`;
- `Результаты`.

Связка:

```text
Ключ → Роль → ROLE-ID → портрет клиента → транскрибация → результат
```

### 7. Мануал должен быть справочником

Вопросы мануала не должны быть просто текстом в промпте.

Нужна сущность:

```text
manual_questions
```

Поля:

- manual_type;
- segment;
- block;
- question_text;
- model_instruction;
- version_date;
- active;
- weight.

### 8. HTML/Excel — выход, не ядро

Отчёт можно скачивать, но он должен собираться из сохранённой карточки.

## Открытые вопросы

### 1. SQLite или Postgres?

Предварительно:

- SQLite — быстрее для MVP;
- Postgres — лучше для долгосрочной системы.

Нужно решить после аудита серверной среды.

### 2. Какой реальный внешний URL Node-версии?

Нужно подтвердить для Codex/server tasks.

### 3. Голосовой бот умеет webhook?

Варианты:

- webhook — лучший;
- JSON/CSV импорт — резерв;
- периодический sync — если webhook невозможен;
- ручная загрузка — аварийно.

### 4. Какие поля обязательны для отчёта?

Предварительно обязательные:

- ФИО;
- base_key;
- сегмент;
- дата тестового дня;
- данные тестового дня;
- хотя бы один источник практики.

Нужно утвердить.

### 5. Какие метрики на трубке обязательны?

Нужно определить:

- минуты;
- количество звонков;
- % звонков >2 минут;
- дни, которые фиксируем;
- целевые значения.

### 6. Как вводить реальные звонки?

Варианты:

- текстом транскрибации;
- файлом;
- ссылкой на аудио;
- ссылкой на систему;
- вручную 3+3+3.

MVP: textarea/file/manual.

### 7. Как получать итоговый тест?

Варианты:

- ручной ввод балла;
- загрузка отчёта;
- ссылка на Testbase/Chatbase/другой сервис;
- импорт CSV/Excel.

### 8. Кто будет пользователем dashboard?

Нужно понять роли:

- Денис;
- тренер;
- наставник;
- HR;
- тимлид;
- админ.

От этого зависят права доступа.

### 9. Нужно ли сохранять старую публичную ссылку?

Предварительно: да, нельзя ломать.

### 10. Какой первый реальный кейс взять для проверки?

Нужен один полный новичок:

- ФИО;
- ключ;
- сегмент;
- сборка;
- маршрут;
- собеседование;
- учебные боты;
- реальные звонки;
- операционка;
- итоговый тест.

На нём проверяется карточка v1.

## Ближайший правильный следующий шаг

Не Codex.

Сначала GPT должен собрать:

```text
Карточка новичка v1 — JSON schema + форма полей + источники + обязательность
```

После этого Codex можно подключать к серверной Phase 1/2.

# Новые решения v2

## 9. Completeness не является аналитикой

`Completeness` показывает только наличие данных.

Он не отвечает на вопрос:

```text
можно ли выпускать новичка?
```

Для этого нужны scores, выводы и рекомендации.

## 10. Оценка должна идти по утверждённым шкалам

Любой ручной или AI-анализ должен использовать документ:

```text
11_EVALUATION_RUBRICS_BY_STAGE_V1.md
```

## 11. Dashboard требует viewer-доступ

Админский ключ нельзя давать всем.

Нужны два слоя:

```text
ADMIN_KEY = сбор и редактирование
VIEWER_KEY = read-only просмотр dashboard и карточек
```

## 12. “Бот учебки” в UI переименовать

В пользовательском интерфейсе использовать:

```text
Учебные агенты
```

`bot_training` остаётся техническим source_code.

## 13. Поле “продукт” в звонках не является критерием оценки

Звонки оцениваются по этапам продаж и мануалу.  
Поле продукта заменить на:

```text
Контекст / тема звонка
```

и сделать необязательным.

## 14. Нужен HTML-отчёт как финальный snapshot

Итоговый отчёт должен храниться на сервере и открываться по ссылке.  
Google Sheets не является финальным отчётом.
