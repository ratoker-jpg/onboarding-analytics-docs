# Phase 3E2 — решения по отчёту, звонкам и учебным агентам

Документ фиксирует продуктовые решения после проверки rich report.

## Главное правило

Отчёт должен разделять три состояния:

- данные загружены;
- данные подготовлены к анализу;
- анализ выполнен и импортирован.

Также отчёт должен разделять источники:

- реальные звонки новичка;
- учебные диалоги с агентами;
- собеседование;
- операционные данные;
- итоговые рекомендации.

## Реальные звонки

Вкладка «Звонки» предназначена только для реальных звонков новичка.

Источники:

- звонки начала;
- звонки середины;
- звонки выпуска.

В этой вкладке не должно быть учебных агентов.

Нужно показывать:

- метрики на трубке;
- список загруженных звонков;
- дату;
- продукт;
- наличие транскрипта;
- комментарий;
- фрагмент транскрипта;
- статус анализа по автомануалу.

Если звонки загружены, но анализ не выполнен, отчёт должен давать действие «Скопировать промпт для Codex».

Если статус показывает «готово», но структурированных звонков нет, отчёт должен объяснять причину.

## Учебные агенты

Учебные агенты не являются реальными звонками.

Верхний уровень вкладки «Учебные агенты» должен показывать не каждую роль отдельно, а результат тренировки:

- Тестовый день;
- Тренировка 2;
- Тренировка 3;
- Тренировка 4.

Первая тренировка всегда называется «Тестовый день».

Для тренировки нужно показывать данные из листа «Результаты»:

- успешно;
- провалено;
- установление контакта;
- выявление потребности;
- презентация продукта;
- работа с возражениями;
- закрытие сделки;
- сегментная оценка;
- комментарии по каждому блоку.

На карточке тренировки нужна кнопка «Посмотреть диалоги».

## Портрет роли

В подробном просмотре диалога нужно подтягивать портрет роли по ID роли.

Поля:

- ID роли;
- ID команды;
- название компании;
- ФИО;
- должность;
- СНО;
- прошлые взаимодействия с клиентом;
- информация о клиенте;
- род деятельности;
- опыт ведения бизнеса.

Для служебного контекста нужна пометка «Служебный контекст роли».

## Оценка учебных агентов

Импортированный результат из бота и оценка Codex должны быть разными блоками.

Возможный будущий тип анализа:

```text
training_agent_analysis_v1
```

Он не должен смешиваться с анализом реальных звонков.

## Операционка

Операционные данные должны получить отдельный будущий анализ.

Возможный будущий тип анализа:

```text
ops_analysis_v1
```

Учитывать:

- статусы;
- просрочки;
- XSales;
- комментарии;
- дополнительные материалы.

## Рекомендации

Итоговые рекомендации должны собираться по всему новичку, а не только по собеседованию.

Учитывать:

- собеседование;
- реальные звонки;
- учебных агентов;
- операционку;
- дисциплину;
- финальный тест;
- красные флаги;
- противоречия между этапами.

Возможный будущий тип анализа:

```text
overall_candidate_recommendation_v1
```

## Мануалы и рубрики

В отчёте должно быть видно:

- какая рубрика использована;
- что именно она оценивает;
- какие блоки входят в оценку;
- какие блоки не были оценены;
- почему оценка частичная;
- где evidence и цитаты.

## Ближайший fix отчёта

Нужно поправить report-v1:

- убрать учебных агентов из вкладки «Звонки»;
- во вкладке «Учебные агенты» показывать результат тренировки верхним уровнем;
- добавить кнопку «Посмотреть диалоги»;
- вывести блоки и комментарии из листа «Результаты»;
- добавить место для портрета роли;
- исправить противоречие «готово, но звонков нет»;
- добавить действие «Скопировать промпт для Codex», если данные есть, а анализа нет.

## Правила следующих PR

- интерфейс на русском;
- технические ключи показывать только как технические подсказки;
- реальные звонки и учебные агенты не смешивать;
- «данные загружены» не равно «анализ выполнен»;
- Codex-балл должен иметь breakdown и evidence;
- если данных недостаточно, это должно быть видно;
- рекомендации собираются по всему профилю новичка.
