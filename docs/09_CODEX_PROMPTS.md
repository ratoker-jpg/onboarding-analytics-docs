# Готовые промпты для Codex

## Важно

Codex используем только для серверной части.

Не давать Codex задачи по Google Sheets, Markdown, структурам или аналитике, если GPT может сделать это сам.

---

# Prompt 1. Server audit only

```text
Ты работаешь с серверным проектом онбординга кросс-продавцов.

ВАЖНО:
Сначала сделай только аудит. Никаких правок, коммитов, деплоев, перезапусков и изменений файлов.
Ответ на русском.
В конце напиши: «Жду команду Делай».

Контекст:
Нужно построить серверную систему аналитики новичков: dashboard, карточка новичка, сравнение новичков, AI-анализ, импорт данных из Google Sheets и голосовых ботов.
Google Sheets не должен быть боевым ядром. Он может быть только import/export/legacy source.

Целевой принцип ключей:
- base_key = человек, например MDLK5Y;
- session_key = конкретный этап/звонок, например MDLK5Y-OB-01, MDLK5Y-EDO-01, MDLK5Y-REAL-S-01.

Проведи аудит проекта через MCP/серверный доступ и ответь:

1. Где лежит активный серверный проект.
2. Какие файлы являются точками входа.
3. Какие есть routes/API/endpoints.
4. Где сейчас хранится состояние новичков, прогресс, результаты и отчёты.
5. Есть ли Google Sheets adapter и как он работает.
6. Есть ли база данных или только snapshot/queue/files.
7. Какие команды можно запускать безопасно для проверки.
8. Какие логи есть и где.
9. Как сейчас устроен deploy/restart.
10. Какие риски при миграции на серверную БД.
11. Какие файлы потенциально придётся менять.
12. Что нельзя трогать.

Формат ответа:
- Короткий вывод.
- Что найдено.
- Текущая архитектура.
- Текущие API.
- Хранилище.
- Google Sheets layer.
- Риски.
- Что можно реализовать.
- Предлагаемая схема данных.
- Предлагаемые API.
- План внедрения.
- Файлы, которые потенциально менять.
- Вопросы к Денису.
- Фраза: «Жду команду Делай».
```

# Prompt 2. Phase 1 — data model design only

```text
Сделай design-only задачу. Никаких правок в коде.

Нужно спроектировать серверную модель данных для Onboarding Analytics Dashboard.

Сущности:
- candidates;
- stage_keys;
- course_progress;
- training_bot_dialogs;
- training_bot_roles;
- voice_bot_sessions;
- real_calls;
- operational_metrics;
- final_tests;
- manual_questions;
- report_snapshots.

Требования:
1. base_key = человек.
2. session_key = конкретный этап/звонок.
3. Данные должны быть append-only там, где это события/диалоги/попытки.
4. Google Sheets — legacy/import/export, не источник истины.
5. Нужно сохранить совместимость с текущим входом по base_key.

Ответь:
- какие таблицы нужны;
- какие поля;
- какие индексы;
- какие связи;
- как мигрировать текущие данные;
- какой storage лучше: SQLite или Postgres;
- какие риски;
- какие endpoints нужны.

Ничего не меняй. Жду команду Делай.
```

# Prompt 3. Phase 2 — implement DB and session keys

```text
Делай только после approval.

Задача:
Добавить серверное хранение candidates/stage_keys и генерацию session_key.

Ограничения:
- не ломать текущий публичный URL;
- не ломать вход по существующему base_key;
- не удалять Google Sheets sync;
- не менять secrets;
- не трогать unrelated UI;
- сделать backup перед правками;
- в конце дать PATCH_REPORT.

Что нужно:
1. Добавить БД или подготовить выбранное хранилище.
2. Создать таблицы candidates и stage_keys.
3. Реализовать генератор session_key формата BASEKEY-STAGE-ATTEMPT.
4. Добавить API:
   - GET /api/candidates/:baseKey
   - POST /api/candidates/:baseKey/stage-keys
   - GET /api/candidates/:baseKey/stage-keys
5. Сохранить совместимость с текущими методами onboarding.
6. Написать минимальные проверки.
7. Дать отчёт:
   - что изменено;
   - какие файлы;
   - как проверить;
   - риски;
   - rollback.
```

# Prompt 4. Import Bot учебки

```text
Audit-only сначала, без правок.

Нужно понять, как безопасно добавить импорт данных из Google Sheet «Бот учебки».

Источники:
1. Лист «Траскрибация» — название с опечаткой, использовать фактическое имя.
   Поля: Ключ, Роль, Дата, Текст.
2. Лист «Роли» — портреты клиентов.
3. Лист «Результаты» — оценки/итоги, если есть.

Логика:
- по base_key найти все строки в «Траскрибация»;
- из колонки «Роль» вытащить ROLE-ID — часть строки до первого `|`;
- по ROLE-ID подтянуть портрет из «Роли»;
- связать с результатом из «Результаты»;
- сохранить как training_bot_dialogs/training_bot_roles.

Ответь:
- какие текущие Google Sheets adapters есть;
- как лучше импортировать;
- какие поля нужны;
- какие API нужны;
- какие риски;
- какие файлы менять;
- как проверить на одном base_key.

Никаких изменений. Жду команду Делай.
```
