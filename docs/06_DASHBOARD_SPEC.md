# Спецификация будущего дашборда

## Цель

Дашборд должен помогать управлять обучением новичков, а не просто показывать красивые графики.

Он должен отвечать на вопросы:

- кто готов к выпуску;
- кто в зоне риска;
- почему человек в риске;
- какие навыки проседают;
- где есть динамика;
- какие этапы обучения работают хуже;
- как новичок выглядит относительно других.

## Главная страница dashboard

### Верхняя панель

Показатели:

- всего новичков;
- в процессе;
- готов к выпуску;
- готов с контролем;
- в зоне риска;
- не рекомендован;
- средний hard score;
- средний soft score;
- средний learning score;
- средний risk score.

### Фильтры

Фильтры:

- поток;
- дата старта;
- сегмент;
- направление;
- наставник;
- рекрутер;
- статус;
- риск;
- этап;
- продукт;
- наличие реальных звонков;
- наличие итогового теста.

## Таблица новичков

Колонки:

- ФИО;

- сегмент;

- дата старта погружения;
- текущий этап;
- hard;
- soft;
- learning;
- discipline;
- risk;
- статус;
- рекомендация;
- дата последнего обновления.

Строка должна быть кликабельной и вести в карточку новичка.

## Визуалы на главном dashboard

### 1. Воронка

```text
Создан → тестовый день → погружение → учебные боты → реальные звонки → итоговый тест → выпуск
```

### 2. Риски

Круговая/столбчатая диаграмма:

- низкий;
- средний;
- высокий;
- критичный.

### 3. Hard vs Soft

Scatter plot:

- X = hard score;
- Y = soft score;
- цвет = risk level;
- точка = новичок.

### 4. Динамика по этапам

Line chart / bar chart:

- тестовый день;
- ОБ;
- ЭДО;
- фонды;
- реальные звонки старт;
- реальные звонки середина;
- реальные звонки финал.

### 5. Heatmap навыков

Строки = новички.

Колонки:

- контакт;
- выявление;
- презентация;
- возражения;
- закрытие;
- сегментная адаптация;
- дисциплина;
- итоговый тест.

## Карточка новичка

Переход по клику на строку.

Вкладки:

1. Обзор.
2. Собеседование.
3. Тестовый день.
4. Погружение.
5. Учебные боты.
6. Реальные звонки.
7. Операционка.
8. Итоговый тест.
9. AI-вывод.
10. История отчётов.

## Сравнение новичков

Пользователь выбирает 2–5 новичков.

Экран показывает:

- общую таблицу;
- bar chart по hard/soft/learning/risk;
- radar chart по навыкам;
- heatmap по этапам;
- текстовое AI-сравнение.

AI-сравнение должно отвечать:

```text
Кто сильнее по hard?
Кто сильнее по soft?
У кого выше риск?
Кто быстрее растёт?
Кого можно выпускать раньше?
Какие разные рекомендации по каждому?
```

## Аналитика по этапам

Экран этапа показывает:

- средний результат;
- количество участников;
- типовые ошибки;
- лучшие результаты;
- худшие результаты;
- корреляцию с итоговым выпуском;
- список новичков, кому нужен повтор.

Примеры этапов:

- тестовый день;
- ОБ;
- ЭДО;
- фонды;
- учебный бот;
- реальные звонки старт;
- реальные звонки середина;
- реальные звонки выпуск;
- итоговый тест.

## Ролевой анализ бота учебки

Отдельный полезный срез:

- какие ROLE-ID чаще приводят к провалам;
- какие типы клиентов сложнее;
- где кандидаты чаще давят;
- где не выявляют потребность;
- какие возражения хуже всего отрабатываются.

## Экспорт

Из карточки и dashboard нужны выгрузки:

- HTML-отчёт по новичку;
- Markdown-отчёт;
- Excel/XLSX выгрузка;
- JSON карточки;
- возможно Google Sheets export для команды.

## MVP dashboard

На первом этапе не надо делать всё.

Минимальный dashboard:

1. список новичков;
2. поиск по ФИО/base_key;
3. фильтр по сегменту/статусу/риску;
4. карточка новичка;
5. hard/soft/learning/risk scores;
6. кнопка “Сформировать отчёт”.

Сравнения и красивые графики — после нормализации данных.

# Обновление v2: dashboard должен быть viewer-слоем

Dashboard не должен использовать `ADMIN_KEY`.

Нужен отдельный read-only доступ:

```text
VIEWER_KEY
```

Viewer может:

- смотреть список новичков;
- смотреть карточку;
- смотреть scores;
- смотреть выводы;
- открывать HTML-отчёты.

Viewer не может:

- создавать кандидатов;
- запускать импорты;
- загружать файлы;
- менять source links;
- редактировать оценки;
- писать в Google Sheets.

## Dashboard показывает аналитику, не completeness

Completeness остаётся техническим полем, но главные поля dashboard:

- hard_score;
- soft_score;
- learning_score;
- discipline_score;
- call_quality_score;
- ops_score;
- final_test_score;
- risk_score;
- overall_score;
- final_status;
- recommendation.
