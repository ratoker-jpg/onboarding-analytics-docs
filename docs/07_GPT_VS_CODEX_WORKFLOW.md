# Рабочее разделение GPT и Codex

## Главный принцип

Codex не жечь на то, что GPT может сделать сам.

Codex используется только там, где нужен доступ к серверу, файловой системе проекта, логам, MCP, деплою или серверному коду.

## Что делает GPT

GPT делает сам:

- анализ Google Drive;
- поиск таблиц и документов;
- чтение Google Sheets через доступы;
- проектирование структуры данных;
- описание модели сущностей;
- подготовку Markdown-документов;
- подготовку ТЗ;
- подготовку промптов для AI;
- подготовку промптов для Codex;
- создание HTML-референсов;
- создание XLSX/Markdown/ZIP-артефактов;
- анализ ручных файлов;
- нормализацию логики;
- составление чек-листов;
- подготовку схемы dashboard;
- разбор методологии;
- формулировки отчётов.

## Что делает Codex

Codex используется только для серверной зоны:

- прочитать активный серверный проект;
- посмотреть реальные файлы на сервере;
- проверить Node routes/API;
- прочитать `.env` без раскрытия секретов;
- проверить logs;
- добавить серверные endpoints;
- подключить SQLite/Postgres;
- написать миграции;
- реализовать генерацию session_key;
- сделать webhook;
- сделать import/export адаптеры;
- перезапустить сервис;
- проверить деплой;
- сделать backup серверных файлов;
- протестировать endpoint.

## Что Codex НЕ должен делать без необходимости

Codex не нужен для:

- анализа Google Sheets, если GPT имеет доступ;
- создания таблиц;
- написания MD-документов;
- описания продуктовой логики;
- ручной аналитики;
- подготовки промптов;
- задач, которые можно сделать локально в чате.

## Стандартный процесс с Codex

### Фаза 1: Audit only

Codex сначала только смотрит.

Запрещено:

- менять файлы;
- коммитить;
- деплоить;
- перезапускать;
- менять `.env`;
- править базу;
- удалять данные.

Ответ должен содержать:

- что найдено;
- какие файлы;
- какие routes;
- где состояние;
- где риски;
- что можно реализовать;
- какие файлы менять;
- план;
- фразу `Жду команду Делай`.

### Фаза 2: Делай

Только после явной команды.

Требования:

- минимальный scope;
- backup перед правками;
- не ломать текущий публичный URL;
- не ломать вход по base_key;
- не ломать текущий тестовый день;
- не ломать текущий отчёт;
- не трогать secrets;
- написать report после правки.

## Что нельзя ломать

- текущую публичную ссылку `/onboarding_cross/`;
- вход новичка по базовому ключу;
- текущий тестовый день;
- текущий HTML-отчёт;
- текущую запись прогресса;
- текущую работу голосового бота;
- Google Sheets-структуру до миграции;
- `.env`, OAuth и Telegram secrets;
- backup/snapshot/queue без плана.

## Как GPT должен ставить задачи Codex

Хорошая задача Codex:

```text
Сначала audit only. Никаких правок.
Проверь серверный проект X.
Нужно понять, как безопасно добавить Y.
Верни список файлов, риски, план и жди команду Делай.
```

Плохая задача Codex:

```text
Сделай дашборд.
```

Почему плохо:

- слишком широкий scope;
- риск сломать текущую систему;
- непонятно, что менять;
- невозможно нормально проверить результат.

## Правило экономии Codex

Если задача про:

```text
понять / описать / спроектировать / собрать документ / создать шаблон / проанализировать таблицу
```

её делает GPT.

Если задача про:

```text
серверный файл / Node / API / БД / deploy / logs / MCP
```

подключается Codex.
