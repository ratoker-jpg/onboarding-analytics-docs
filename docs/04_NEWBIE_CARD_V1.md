# Карточка новичка v1

## Цель карточки

Карточка новичка должна отвечать на вопросы:

1. Кто этот новичок?
2. Как он показал себя до выхода в обучение?
3. Как прошёл тестовый день?
4. Как проходил погружение?
5. Как справился с учебными ботами?
6. Как ведёт реальные звонки?
7. Какая динамика?
8. Где hard/soft риски?
9. Что тренеру делать дальше?
10. Можно ли выпускать?

## Экран формы сборки

На первом этапе нужна не красивая аналитика, а форма, которая собирает нормализованные данные.

Рабочий сценарий:

```text
Админ / GPT / тренер открывает форму
→ вводит base_key
→ система подтягивает всё, что может автоматически
→ пользователь добавляет ручные данные
→ нажимает «Сформировать отчёт»
→ AI формирует портрет
→ результат сохраняется в базу
```

## Блок 1. Основное

Поля:

| Поле | Тип | Источник | Обязательное |
|---|---|---|---|
| ФИО | текст | авто/ручной | да |
| base_key | текст | авто/ручной | да |
| сегмент | выбор | авто/ручной | да |
| направление | выбор | авто/ручной | желательно |
| дата тестового дня | дата | авто/ручной | да |
| дата старта погружения | дата | авто/ручной | желательно |
| наставник | текст | ручной | желательно |
| рекрутер | текст | авто/ручной | желательно |
| статус | выбор | авто | да |

## Блок 2. Собеседование

Поля:

| Поле | Тип | Источник |
|---|---|---|
| дата собеседования | дата | `Кроссы подбор / Собесы` |
| роль | текст | `Кроссы подбор` |
| рекрутер | текст | `Кроссы подбор` |
| транскрибация собеседования | textarea / файл | ручной ввод |
| комментарий HR | textarea | ручной ввод |
| комментарий TL | textarea | ручной ввод |
| предварительный риск | выбор | ручной/AI |

AI должен оценить:

- soft-профиль;
- мотивацию;
- структурность мышления;
- обучаемость;
- потенциальный риск;
- вопросы для тренера.

## Блок 3. Тестовый день

Источники:

- `Веб-версия МВП / Сборка`;
- результаты голосового бота;
- тесты;
- открытые ответы;
- питчи до/после обратной связи.

Подтягивать:

- тест по тарифам;
- тест по этапам продаж;
- тест по СНО;
- тест по ОБ;
- ответ по этапам;
- ответ по СНО;
- рефлексию по прослушке;
- питчи до/после ОС;
- результаты голосового бота;
- вывод.

AI должен оценить:

- теорию;
- практику;
- способность принять ОС;
- динамику до/после;
- базовый hard-риск.

## Блок 4. Маршрут погружения

Источник:

```text
Маршрут новичка — кросс-продавец v1
```

Подтягивать:

- какие дни открыты;
- какие дни завершены;
- какие блоки завершены;
- какие материалы открывал;
- где были задержки;
- вопросы помощи;
- логи прохождения;
- сводку новичка.

AI должен оценить:

- дисциплину обучения;
- самостоятельность;
- темп прохождения;
- интерес/вовлечённость;
- тревожные сигналы.

## Блок 5. Учебные диалоги с ботом

Источник:

```text
Бот учебки / Траскрибация + Роли + Результаты
```

Автоматическая логика:

1. По `base_key` найти все строки в `Траскрибация`.
2. Из колонки `Роль` достать `ROLE-ID`.
3. По `ROLE-ID` найти портрет клиента в `Роли`.
4. Подтянуть результат из `Результаты`, если есть.
5. Сформировать список учебных диалогов.

Для каждого диалога:

- role_id;
- дата;
- клиент;
- компания;
- должность;
- СНО;
- род деятельности;
- взаимодействия ранее;
- информация о клиенте;
- критерий успеха;
- критерий провала;
- целевое действие;
- ожидаемые возражения;
- тон клиента;
- транскрибация;
- результат;
- AI-разбор.

## Блок 6. Реальные звонки

Пока преимущественно ручной ввод/загрузка.

Группы:

```text
3 звонка начала
3 звонка середины
3 звонка выпуска
```

Для каждого звонка:

| Поле | Тип |
|---|---|
| период | start/middle/final |
| дата | дата |
| ссылка на аудио | URL |
| транскрибация | textarea/file |
| сегмент клиента | выбор |
| продукт | выбор |
| результат звонка | выбор |
| комментарий тренера | textarea |

AI должен оценить по мануалу:

- контакт;
- выявление;
- презентация;
- возражения;
- закрытие;
- сегментная адаптация;
- риски;
- динамику между периодами.

## Блок 7. Операционка

Ручной ввод на MVP.

Поля:

- день обучения;
- дата;
- минут на трубке;
- количество звонков;
- % звонков больше 2 минут;
- качество заполнения целей;
- наличие просроченных целей;
- качество статусов;
- комментарий;
- дисциплина;
- самостоятельность.

Пример:

```text
День 3 — 120 минут, 31% звонков >2 мин, цели ведёт нестабильно
День 4 — 180 минут, 37% звонков >2 мин, статусы улучшились
День 5 — 210 минут, 41% звонков >2 мин, готов к выпуску с контролем
```

## Блок 8. Итоговый тест

Поля:

- название теста;
- балл;
- максимум;
- процент;
- темы ошибок;
- вопросы с ошибками;
- ссылка/файл отчёта;
- комментарий;
- AI-вывод.

AI должен определить:

- что западает;
- насколько ошибки критичны;
- связано ли это с реальными звонками;
- какие темы повторить.

## Блок 9. Итоговая AI-оценка

На выходе карточка должна иметь:

- `hard_score`;
- `soft_score`;
- `learning_score`;
- `discipline_score`;
- `risk_score`;
- `risk_level`;
- `strengths`;
- `growth_zones`;
- `red_flags`;
- `coach_recommendations`;
- `final_decision`.

## Итоговые решения

Возможные статусы:

| Статус | Значение |
|---|---|
| `ready` | готов к выпуску |
| `ready_with_control` | готов, но нужен контроль |
| `needs_practice` | нужна дополнительная практика |
| `high_risk` | высокий риск, выпускать рано |
| `not_recommended` | не рекомендован |

## Минимально обязательный набор для отчёта

Без этого отчёт лучше не формировать:

- ФИО;
- base_key;
- сегмент;
- дата тестового дня;
- данные тестового дня / сборки;
- хотя бы один источник практики: бот или реальные звонки.

Желательные данные:

- собеседование;
- маршрут погружения;
- реальные звонки старт/середина/финал;
- операционка;
- итоговый тест.
